TL;DR

Kimi AI shared its scaling methodology for open frontier models, detailing advancements in token efficiency via the Muon optimizer, long-context handling through the KimiLinear architecture, and multi-agent coordination using agent swarms. These innovations culminated in the Kimi K2.5 model and previewed next-generation architectures like Attention Residue.

Token Efficiency & The Muon Optimizer

  • Token Efficiency as Intelligence: Zhilin Yang emphasized that token efficiency does more than save infrastructure costs; by virtually multiplying limited high-quality data (e.g., turning 50 trillion tokens into 100 trillion), it raises the upper bound of model intelligence.
  • Muon Optimizer Integration:
    • Replaced traditional AdamW with the second-order Muon optimizer, where every gradient update is orthogonalized, yielding a 2× token efficiency improvement.
    • Employed critical techniques including weight decay and adjustable RMS update coefficients to match Adam's consistency.
    • Developed a distributed Muon implementation that partitions states across data-parallel Nvidia GPU clusters to ensure memory efficiency.
  • QK Clip for Stability at Scale:
    • Scaling Muon to a 1-parameter trillion model initially caused training instability and max logits exceeding 1,000.
    • Introduced QK Clip, computing max logits per attention head during the forward pass and applying a dividing factor to key and query projections to constrain values without harming convergence.

Long-Context Scaling & KimiLinear Architecture

  • Limitations of Standard Global Memory: Traditional linear attention uses a single global decay factor, forcing a binary choice between retaining everything or forgetting past context.
  • KimiLinear & KimiDelta Attention:
    • Introduced a fine-grained diagonal matrix decay factor ($\alpha$) per channel, allowing some channels to retain long-range context while others quickly forget and refresh.
    • Utilized an exact, mathematically equivalent chunk-wise formulation to parallelize computation on modern GPUs without approximation loss.
    • Mixed linear and full attention layers in a 1:2:3 ratio to balance long-context capabilities with implementation efficiency.
  • Performance Benchmark: KimiLinear outperformed both MLA and GDM variants on short-context tasks (MMLU) and long-context tasks (Ruler), marking an architecture capable of beating full attention across short context, long inputs, and long outputs.

Agent Swarms & Learning Paradigms

  • Agent Swarm Architecture: Moving beyond single-agent setups, Kimi introduced an orchestrator (main agent) that spawns and manages parallel sub-agents (e.g., researchers, web developers, fact-checkers) to tackle complex, multi-step tasks.
  • Specialized RL Reward Objectives:
    • Instantiation Reward: Incentivizes sub-agent instantiation to prevent serial collapse into single-agent execution (decayed over training).
    • Finished Reward: Ensures created sub-tasks maintain a high completion ratio rather than spawning meaningless pseudo-tasks.
    • Outcome Reward: Measures whether the overall task is successfully completed.

Kimi K2.5: Native Joint Vision-Text Training

  • Stable Base Training: The K2.5 base model was trained smoothly over 15 trillion tokens (plus an additional 15 trillion tokens), running on Nvidia H100 GPU clusters (each node featuring 2TB RAM and 8 GPUs connected via NVLink) with zero loss spikes, aided by the Muon optimizer.
  • Early Fusion Approach: Unlike models that stack vision onto an already-trained text base, Kimi K2.5 fuses vision and text tokens from day one (0% progress) in a shared representation space, enabling capabilities like direct vision-to-code generation.
  • Cross-Modality Enhancement:
    • Vision RL (training exclusively on vision tasks) improved performance on heavy text tasks.
    • Zero Vision SFT was utilized: the model used zero vision SFT data and only text SFT data during joint alignment, yet achieved state-of-the-art performance on vision tasks.

Preview: Attention Residue Architecture

  • Depth-Dimension Attention: Drawing an analogy between standard residual connections and LSTM (viewed as a 90-degree rotated recurrent net), Kimi introduced Attention Residue, which collects all previous hidden states across layers and uses an attention mechanism to compute the current layer's state.
  • Block Attention Residue Variant: To minimize communication and memory overhead, layers are divided into blocks (e.g., 4 to 16 layers per block) where attention residue is applied at the block level, while standard residuals are kept within blocks.
  • Results: Improved token efficiency by 24% and demonstrated notable gains on coding, math, and reasoning benchmarks including GPQA, math, and HumanEval.
